Overview

The Pokedex API lets users view, filter, add, and edit Pokemon data. It serves JSON responses.

Endpoints

1. GET /getPokemon

Methods: GET, HEAD

Returns all Pokemon. Optional query parameters allow for type and weakness filtering.

Query Parameters

Name Type Required Description
type string optional Filter by Pokemon type
weakness string optional Filter by Pokemon weakness

Example Request

GET /getPokemon?type=Fire&weakness=Water

Example Response (200 OK)

{
  "pokemon": [
    {
      "id": "004",
      "name": "Charmander",
      "type": ["Fire"],
      "weaknesses": ["Water","Ground"],
      "height": "0.6 m",
      "weight": "8.5 kg"
    }
  ]
}

2. POST /addPokemon

Methods: POST

Adds a new Pokemon. JSON body is required.

Body Parameters

Name Type Required
id string yes
name string yes
type array of strings no
weaknesses array of strings no
height string no
weight string no

Example Request

POST /addPokemon
Content-Type: application/json

{
  "id": "004",
  "name": "Charmander",
  "type": ["Fire"],
  "weaknesses": ["Water","Ground"],
  "height": "0.6 m",
  "weight": "8.5 kg"
}

Example Response (201 Created)

{
  "message": "Created Successfully",
  "pokemon": {
    "id": "004",
    "name": "Charmander",
    "type": ["Fire"],
    "weaknesses": ["Water","Ground"],
    "height": "0.6 m",
    "weight": "8.5 kg"
  }
}

3. PUT /editPokemon

Methods: PUT

Updates an existing Pokemon by ID. JSON body required.

Body Parameters

Name Type Required
id string yes
name string yes
type array of strings no
weaknesses array of strings no
height string no
weight string no

Example Request

PUT /editPokemon
Content-Type: application/json

{
  "id": "004",
  "name": "Charmeleon",
  "type": ["Fire"],
  "weaknesses": ["Water","Ground"],
  "height": "1.1 m",
  "weight": "19.0 kg"
}

Example Response (200 OK)

{
  "message": "Updated Successfully",
  "pokemon": {
    "id": "004",
    "name": "Charmeleon",
    "type": ["Fire"],
    "weaknesses": ["Water","Ground"],
    "height": "1.1 m",
    "weight": "19.0 kg"
  }
}

4. HEAD /getPokemon

Methods: HEAD

Returns metadata headers only. No body content.

5. Invalid Endpoints

Any unknown route returns 404 Not Found.

{
  "message": "The page you are looking for was not found."
}
